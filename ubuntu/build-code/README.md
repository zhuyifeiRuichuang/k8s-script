# 说明
使用docker进行代码打包。推荐使用最新版本docker。  
优点：本地无残留，对本地环境无干扰。让开发专注于提交代码做开发，无需额外配置环境。  
缺点：依赖docker，每次都要全新构建环境，依赖网络。  
注意：一个代码目录只能同时运行一个打包脚本，否则会产生读写冲突。  

## 文件说明
| 文件 | 说明 |
|----|----|
| java-jar.sh | 将Java代码打包为jar文件，可修改脚本中依赖软件版本。推荐放在代码根目录执行。 |
| web-dist.sh | 打包前端代码，生成dist目录。可修改脚本中依赖软件版本。将脚本放在代码目录的根目录执行。 |
## 使用方法
修改脚本中依赖软件版本。将脚本放在代码根目录，`bash *.sh` 执行脚本，自动打包。
